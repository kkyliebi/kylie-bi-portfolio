# Portfolio Website
## Creative Direction & Communication Method Blueprint

---

# Purpose

This document defines the highest-level creative direction for the portfolio website.

It serves as the primary guideline governing every design, content, interaction, and implementation decision throughout the project.

Whenever implementation choices conflict with local optimisations, this document takes priority.

The goal of this website is not feature completeness, but professional coherence.

Every decision should reinforce one unified communication methodology.

---

# Project Overview

This portfolio website is not a traditional design portfolio.

It is a long-term professional communication platform representing the methodology, thinking process, and creative identity of Kylie Bi.

The website should function as a curated communication archive rather than a collection of unrelated projects.

Every project contributes to one coherent professional narrative.

The portfolio itself should become an example of communication design.

---

# Core Design Principle

The website should never answer:

> What did Kylie design?

Instead, it should gradually answer:

> How does Kylie think?

Visitors should leave understanding the methodology behind the work rather than remembering isolated visual outcomes.

---

# Professional Positioning

Communication Designer & Creative Producer

working across

- Visual Communication
- Creative Direction
- Concept Development
- Narrative Systems
- Creative Production
- Brand Communication
- Interdisciplinary Storytelling

The website should never position Kylie primarily as a graphic designer.

Instead, it should communicate a hybrid creative practice connecting research, communication strategy, visual systems, and creative production.

---

# Core Professional Value

The central professional value is:

> Transforming abstract concepts into narrative-driven communication systems.

This statement should remain the conceptual foundation of the entire website.

Every page should reinforce it.

---

# Design Philosophy

Design is not defined by visual output.

Visual language is one way ideas become understandable.

The problem determines the medium.

Every project should demonstrate:

- why the problem matters
- how relationships are discovered
- how concepts are translated
- how communication systems are constructed
- how ideas become tangible experiences

Thinking always precedes form.

---

# Creative Methodology

The entire website should quietly embody the following methodology.

Discover

↓

Connect

↓

Translate

↓

Materialise

Visitors do not need to explicitly read these words.

Instead, they should naturally experience this progression through navigation, pacing, storytelling, and interaction.

This methodology should influence:

- Information architecture
- Navigation
- Page transitions
- Typography hierarchy
- Spacing
- Visual rhythm
- Interaction behaviour
- Project sequencing

---

- a Behance portfolio

- a commercial agency website

# Motion Philosophy

Motion exists to support communication.

Not decoration.

Transitions should feel:

- calm
- precise
- architectural
- slow
- deliberate

Nothing should move simply because it can.

Every movement should clarify structure.

---

# Typography

Typography establishes narrative hierarchy.

Typography should create rhythm.

Not decoration.

Large negative space is encouraged.

Silence is part of communication.

---

# Image System

Images function as documentary evidence rather than decoration.

Maintain consistency across:

- camera language
- lighting
- materials
- composition
- perspective
- colour palette

The website should feel as if photographed by one photographer under one creative direction.

---

# Non-goals

The project is NOT trying to become:

- a trendy portfolio
- a startup landing page
- an Apple-style website
- a highly animated showcase
- a commercial agency website
- a UI gallery
- a Behance clone
- a student graduation portfolio

Avoid introducing features simply because they are visually impressive.

Every addition must strengthen communication rather than increase complexity.

---

# Decision Framework

Before implementing any change, evaluate it against the following questions.

1. Does it reinforce the communication methodology?

2. Does it strengthen professional coherence?

3. Does it improve editorial clarity?

4. Does it help visitors understand the thinking process?

5. Would a European creative studio consider this decision timeless?

If any answer is "No",

the implementation should be reconsidered.

---

# Codex Working Principles

When proposing any implementation, always prioritise:

1. Professional Coherence

2. Methodological Coherence

3. Editorial Clarity

4. Narrative Sequencing

5. Communication Effectiveness

6. Simplicity

7. Timelessness

Never optimise only for visual novelty.

Always ask:

"Does this decision better communicate the methodology?"

If not,

reconsider the solution.

---
