# 1. Purpose

This document defines the visual identity of the portfolio website.

It answers one question:

> **What should the website look and feel like?**

It does **not** define implementation details or component specifications.

Implementation belongs to `DESIGN_TOKENS.md`.

Component selection belongs to `COMPONENT_RULES.md`.

---

# 2. Overall Identity

The portfolio should feel like:

- A curated editorial archive
- A communication cases archive
- A contemporary European design publication
- A museum documentation system
- A long-term knowledge collection

It should **never** feel like:

- A startup landing page
- A SaaS product website
- A marketing agency portfolio
- A template showcase
- A personal blog

# 4. Design Personality

The website should communicate:

- Confidence without showing off
- Clarity over decoration
- Thinking before aesthetics
- Systems before objects
- Communication before graphics

The visual language should feel calm, intentional and highly controlled.

---

# 5. Layout Language

The overall layout follows an editorial logic.

Characteristics:

- Horizontal storytelling
- Large breathing spaces
- Clear reading rhythm
- Strong hierarchy
- Asymmetrical but balanced
- Typography-driven composition
- Modular information blocks

Avoid:

- Dense dashboards
- Marketing sections
- Overlapping decorative elements
- Card-heavy interfaces
- Visual clutter

---

# 6. Typography Language

Typography is the primary visual medium.

Text is content, navigation and composition at the same time.

Characteristics:

- Large headings
- Comfortable reading width
- Strong hierarchy
- Editorial spacing
- Minimal emphasis
- Consistent rhythm

Typography should create structure before images do.

---

# 7. Color Language

Overall palette:

- Neutral
- Quiet
- Low saturation
- High readability

General direction:

- White / Off-white backgrounds
- Dark grey typography
- Limited accent usage

The interface should never rely on color for visual excitement.

---

# 8. Image Language

Images should support the narrative rather than dominate it.

Preferred characteristics:

- Editorial photography
- Museum documentation
- Architectural compositions
- Material-focused close-ups
- Soft natural lighting
- Neutral backgrounds
- Matte surfaces
- Minimal visual noise

Avoid:

- Stock-photo aesthetics
- Hyper-commercial imagery
- Neon palettes
- Excessive CGI rendering
- Decorative AI artwork
- Visual effects without purpose

---

# 9. Motion Language

Motion exists to improve reading and orientation.

Its purpose is:

- Reveal
- Transition
- Guide attention
- Maintain narrative flow

Motion should never become the visual focus.

Avoid:

- Bounce
- Elastic animations
- Flashy transitions
- Decorative motion
- Attention-seeking effects

---

# 10. Information Density

Information should feel curated rather than compressed.

Each section should present:

- One primary idea
- Supporting evidence
- Clear visual hierarchy

Whitespace is treated as part of the content.

---

# 11. Visual Consistency

Every page should feel like it belongs to the same publication.

Consistency applies to:

- Layout rhythm
- Typography
- Image treatment
- Caption style
- Spacing
- Navigation
- Motion
- Graphic language

Consistency is more important than visual variety.

---

# 12. Visual Evaluation Principles

Every visual decision should be evaluated against the following questions:

1. Does it strengthen the editorial identity?
2. Does it support the archive concept?
3. Does it improve narrative clarity?
4. Does it reduce unnecessary visual noise?
5. Does it remain timeless rather than trendy?
6. Does it feel consistent with the rest of the website?

If the answer to multiple questions is **No**, the solution should be reconsidered.
